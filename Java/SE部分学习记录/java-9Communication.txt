类与类之间的关系：继承，关联
    ①继承：is-a关系-是
    ②关联：has-a关系-有  一个类被另一个类作为属性叫做封装时构成的关系叫做关联（代码体现就是B为A的属性存在）
    ③依赖：use-a关系-依靠  一个类A中的方法的执行需要依靠另一个类B的支持，此时构成依赖关系
          (代码体现，一个类B存在于类A方法的参数区)
工厂模式：
单例模式：保证这个构造方法私有化，创建一个提供本地实例的方法（分为饿汉和饱汉方式，前者赋值在属性区，后者使用方法创建）
多例模式：保证构造方法私有化，创建一个提供本类不同实例的方法

enum与Enum的区别：enum是声明枚举类型的关键字，Enum是java.lang包中的一个类，enum声明后，相当于一个类从Enum中去继承。

泛型：实质是参数化类型（动态的确定当前操作的类型）
	泛型作用：简单通用，增加安全性，可以直接的进行类型转换
	泛型类，泛型方法，泛型接口
	例：public class M<T>{//泛型类
	}
